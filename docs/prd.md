1. 产品概览

1.1 产品定位
一个帮助大学生进行自我职业探索的平台，核心通过圆桌讨论的方式让学生发现职业兴趣、提升认知并找到"同路人"，同时提供招聘活动日历作为辅助信息。

1.2 核心目标
- 帮助学生了解自己的职业兴趣和能力边界
- 提供真实的职业信息和场景模拟
- 连接志同道合的学生社群
- 培养学生的职业规划思维
1.3 目标用户
对职业规划有初步需求但缺乏方向的大学生

1.4 产品版本
- 版本号：V1.0.0
- 版本名称：MVP1.0版本
- 发布日期：待定

3. 核心流程详解

3.1 用户首次使用流程

flowchart TD
    A[用户进入导入页] --> B{是否已登录?}
    B -->|否| C[显示导入页]
    C --> D[点击注册/登录]
    D --> E[注册/登录成功]
    E --> F{是否已设置个性化偏好?}
    F -->|否| G[进入个性化设置页]
    G --> H[设置地点/行业/企业规模/加班偏好]
    H --> I[点击保存并进入]
    I --> J[保存偏好设置]
    J --> K[进入首页-日历]
    F -->|是| K
    B -->|是| K

流程说明：
1. 用户进入导入页，了解平台定位
2. 已登录用户直接跳转到首页
3. 未登录用户点击注册/登录
4. 注册/登录成功后，判断用户是否已设置个性化偏好
5. 未设置偏好的用户进入个性化设置页
6. 用户设置地点、行业、企业规模、加班偏好（所有选项必须至少选择一项）
7. 点击"保存并进入"，保存偏好设置
8. 进入首页-日历页面
3.2 圆桌讨论参与流程

flowchart TD

    I --> J[群聊满6人推送小组长确认消息]

3. 系统自动分配到未满员的小组，每满6人自动成组并创建群聊
4. 根据邀请码创建圆桌会议群聊，满6人推送小组长确认消息，第一个确认者成为组长，12小时内无人确认则随机指定
5. 组长点击“分享日历邀请”，5个组员同意后，组长可以随时点击“共享日历”按钮查看
6. 组长点击"发起会议"根据成员日历空闲时段（只显示已约/空闲状态，保护隐私）填写会议事件信息，选择会议时间、填写会议链接/地址、备注，系统校验时间不冲突
7. 创建会议成功，推送会议公告到群聊，同步会议到所有成员日历
8. 小组成员在会议前完成"摸索认知边界"问题清单，形成原始认知报告
9. 参与2小时圆桌讨论
10. 讨论结束后，每个人更新认知报告
11. 更新个人事件状态为已完成

4. 业务规则与逻辑详解

4.1 用户注册与登录

注册规则：
- 邮箱格式校验：必须符合邮箱格式（xxx@xxx.xxx）
- 密码长度校验：至少8位字符
- 邮箱唯一性校验：邮箱不能已被注册
- 密码加密存储：使用 bcrypt 等加密算法
登录规则：
- 邮箱和密码校验：必须匹配数据库记录
- Token生成：登录成功后生成 JWT Token
- Token有效期：7天
- 登录状态判断：根据 Token 判断用户登录状态
忘记密码规则：
- 邮箱校验：必须存在于数据库
- 重置链接：发送重置密码链接到用户邮箱
- 链接有效期：24小时
4.2 个性化偏好设置

设置规则：
- 必填校验：所有选项必须至少选择一项
- 多选支持：所有选项支持多选
- 城市选项：60+主要城市
- 行业选项：16个主要行业
- 企业规模选项：5个等级
- 加班偏好选项：7种工作制
修改规则：
- MVP版本：个性化偏好设置后不可修改
- 后续版本：可能开放修改功能
4.3 招聘活动管理

活动展示规则：
- 月视图展示：按月展示活动
- 活动数量标识：日历上显示每天的活动数量
- 企业类型颜色：国企[浅灰]、外企[浅紫]、民企[浅黄]、商业雇主[浅蓝]、非营利组织[浅绿]
- 活动状态：正常、已取消
用户操作规则：
- 申请：提交申请，等待管理员审核
- 感兴趣：标记感兴趣状态
- 已参加：申请通过后可标记已参加
- 收藏：收藏活动便于后续查找
4.4 圆桌讨论管理

组队规则：
- 自动分配：系统自动按邀请码分配
- 满员推举组长：每满6人需确认组长
小组长规则：
- 确认机制：群聊推送确认消息，第一个确认者成为组长
- 超时机制：12小时内无人确认则随机指定组长
- 组长权限：只有组长可以发起会议
会议规则：
- 时间协调：组长查看成员日历空闲时段
- 时间冲突校验：会议时间必须与所有成员已约时间不冲突
- 会议同步：会议事件自动同步到所有成员日历
- 会议创建：填写时间、链接/地址、备注
问题清单规则：
- 会前准备：成员需要在会议前完成问题清单
- 完成状态：显示已完成和未完成的成员
- 问题内容：固定3个问题（MVP版本）
完成标记规则：
- 单独标记：每个人单独标记讨论完成
- 状态更新：标记完成后更新个人事件状态
- 小组完成：所有成员标记完成后，小组状态更新为已完成
4.5 通知系统

通知类型：
- 新活动提醒：新增符合用户偏好的活动时推送
- 申请状态更新：用户申请的活动状态变更时推送
- 圆桌讨论邀请：用户被邀请参加圆桌讨论时推送
- 会议时间提醒：圆桌讨论会议即将开始时推送（提前1小时）
通知展示规则：
- 未读标识：未读消息显示红点提示
- 已读标识：已读消息不再显示红点
- 通知内容：包含标题、时间、摘要
- 通知排序：按时间倒序排列
4.6 隐私保护

学生证隐私：
- 使用范围：学生证照片仅用于身份验证
- 展示范围：不公开显示，仅用户本人可见
- 存储安全：加密存储，防止泄露

6. 产品亮点

1. 核心价值明确：聚焦大学生的自我职业探索需求，通过"能/不能"对比表清晰传达平台定位。

2. 圆桌讨论机制：创新的6人小组模式，通过线上/线下讨论帮助学生找到"同路人"。

3. 智能化匹配：基于个性化偏好设置，为学生推荐精准的招聘活动和圆桌讨论。

4. 日历可视化：按月展示活动，按企业类型颜色区分，直观清晰。

5. 隐私保护：小组成员只能查看彼此的日历空闲状态，保护个人隐私。

6. 会前准备：通过"摸索认知边界"问题清单，引导学生深度思考，提升讨论质量。
