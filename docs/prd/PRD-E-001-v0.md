## 体验北极星

> **一句话**: 让迷茫的学生在3分钟内看到一个"与自己相关"的职业世界入口，并在7天内完成一次认知突破。

### 体验原则
1. **零废话**: 每一步都必须有价值增量，不填无用表单
2. **可感知的进步**: 每次操作都能看到"认知边界"在扩展
3. **人比信息重要**: 圆桌讨论是核心，日历是入口，不是反过来

---

## 1. 背景与动机

### 现状
- 二本三本学生缺乏职业规划指导资源
- 招聘信息分散，学生不知如何筛选
- 同辈群体认知同质化，缺乏多元视角碰撞

### 问题
1. **信息黑洞**: 不知道有什么机会适合自己
2. **认知孤岛**: 不知道自己真正想要什么
3. **路径迷雾**: 不知道从现在到目标该怎么走

### 影响
- 错过适合的招聘机会
- 盲目投递，效率低下
- 入职后发现与预期不符，离职率高

## 2. 目标与成功标准

### 业务目标（引用 biz-overview）
- 帮助学生看清职业真相，自主找到职业路径

### 成功指标（MVP 验证期）

| 指标 | 目标 | 口径说明 |
|-----|-----|---------|
| 注册转化率 | ≥90% | 完成注册人数 / 访问注册页人数 |
| 日历访问率 | ≥90% | 注册后7天内访问日历 / 注册人数 |
| 圆桌报名率 | ≥90% | 访问日历后报名圆桌 / 访问日历人数 |
| 圆桌到场率 | ≥90% | 实际参与圆桌 / 报名人数 |
| 次日留存率 | ≥90% | 次日活跃 / 当日新增注册 |
| 7日留存率 | ≥90% | 第7日活跃 / 第1日新增注册 |

### 止损信号
- 任意核心指标连续3天低于70%
- 用户反馈"不知道这APP干嘛的"占比 > 20%

## 4. 用户与场景

### 目标用户画像

**主画像: 迷茫的大三学生小王**
- 二本院校，专业不热门
- 身边同学都在考研/考公，但自己不确定
- 刷招聘APP但不知道投什么
- 想了解真实职场，但缺乏渠道

### 关键场景

#### S1: 注册引导
> 晚上宿舍床上，刷到推广链接 → 3分钟完成注册 → 看到与自己相关的日历

#### S2: 日历探索
> 打开APP，看到本月有哪些招聘活动 → 按自己偏好筛选 → 标记感兴趣的活动

#### S3: 圆桌报名
> 收到匹配通知 → 查看小组成员画像 → 报名参与

#### S4: 圆桌讨论
> 进入会议室 → 跟着引导问题讨论 → 看到自己的认知边界图

#### S5: 认知对比
> 讨论结束 → 查看自己的认知边界 → 与他人对比 → 发现新视角

## 5. 功能需求

### F1: 学生端首页

#### F1.1 注册/登录
- 手机号 + 验证码登录
- 新用户自动创建账号
- 验证码60秒有效期

**业务规则**:
- 一个手机号只能绑定一个账号
- 验证码错误3次锁定15分钟
- 支持重新发送验证码（60秒后）

#### F1.2 学生证OCR
- 拍照识别学生证
- 自动填充: 学校、专业、年级、姓名
- 用户可修改OCR结果

**边界条件**:
- OCR失败 → 手动输入引导
- 学生证模糊 → 提示重新拍照
- 非学生证 → 提示上传正确证件

#### F1.3 个人资料
- 必填: 昵称、学校、专业、年级、毕业年份
- 选填: 13维度偏好设置
- 隐私: 手机号仅自己可见

### F2: 日历核心

#### F2.1 13维度筛选

| 维度 | 说明 | 选项示例 |
|-----|-----|---------|
| 地点 | 工作城市 | 一线城市/新一线/二线/其他 |
| 自我定位 | 职业角色倾向 | 技术/产品/运营/销售/职能 |
| 发展方向 | 职业发展路径 | 专家型/管理型/创业型 |
| 行业 | 目标行业 | 互联网/金融/制造业/教育... |
| 平台性质 | 企业类型 | 国企/外企/民企/创业公司 |
| 企业规模 | 公司人数 | 100人以下/100-500/500-2000/2000+ |
| 企业文化 | 工作氛围 | 扁平/层级/结果导向/过程导向 |
| 领导风格 | 期望的管理方式 | 放手型/指导型/导师型 |
| 培训项目 | 是否看重培训 | 是/否/无所谓 |
| 加班偏好 | 加班接受度 | 接受/不接受/看情况 |
| 假期 | 假期要求 | 双休/单休/弹性 |
| 医疗保障 | 医疗福利要求 | 有/无/无所谓 |
| 生育福利 | 生育相关福利 | 有/无/无所谓 |

#### F2.2 月份视图
- 展示当月所有招聘活动
- 按日期排列
- 点击查看活动详情

#### F2.3 颜色编码

| 企业类型 | 颜色 |
|---------|------|
| 国企 | 灰色 #9E9E9E |
| 外企 | 紫色 #9C27B0 |
| 民企 | 黄色 #FFC107 |
| 创业公司 | 橙色 #FF9800 |
| 其他 | 蓝色 #2196F3 |

### F3: 圆桌讨论

#### F3.1 相似度匹配
- 基于13维度偏好计算相似度
- 匹配6人小组（相似但非完全相同）
- 显示匹配度百分比

**匹配规则**:
```
相似度 = 相同维度数 / 13 * 100%
匹配阈值: 40%-70%（太相似无碰撞，太不同无法共鸣）
```

#### F3.2 会议流程引导
- 20个核心问题清单
- 问题逐个展示
- 每个问题3-5分钟讨论时间
- 主持人引导（第一版可人工）

**20个核心问题示例**:
1. 你为什么想进入这个行业？
2. 你对这个行业最大的误解是什么？
3. 你最担心入职后遇到什么问题？
4. ...（完整清单待确认）

#### F3.3 实时通信
- 语音通话（主）
- 文字聊天（辅）
- 支持静音、举手发言

**技术要求**:
- 语音延迟 < 200ms
- 支持6人同时在线
- 弱网自动降级为文字

### F4: 认知边界图

#### F4.1 雷达图展示
- 维度: 行业认知、岗位认知、能力认知、路径认知、资源认知
- 覆盖面积 = 认知广度
- 颜色深度 = 认知深度（来源质量）

**认知深度颜色**:
- 深色: 亲身经历
- 中色: 他人分享
- 浅色: 网络信息

#### F4.2 小组对比
- 叠加显示小组成员边界
- 差异区域高亮
- 点击查看他人信息来源

#### F4.3 历史轨迹（Should）
- 展示认知边界变化
- 标记关键事件（参与圆桌、阅读文章等）

### F5: 通知系统

#### F5.1 活动提醒
- 提前3天推送
- 提前1天推送
- 当天推送

#### F5.2 圆桌通知
- 匹配成功通知
- 会议开始前15分钟提醒
- 会议开始通知

## 8. 业务约束与风险

### 用户体验要求
- 注册流程不超过3步
- 首屏加载时间 < 2秒
- 日历筛选响应时间 < 500ms

### 错误处理
- 网络错误: 友好提示 + 重试按钮
- 匹配失败: 显示原因 + 建议调整偏好
- 会议掉线: 自动重连 + 提示

### 权限和隐私
- 手机号仅用于登录，不对外展示
- 学生证照片不上传服务器（仅本地OCR）
- 用户可删除账号及所有数据

### 业务约束
- 每人每周最多参与2次圆桌
- 圆桌最少4人开场，最多6人
- 匹配等待超时（24小时）自动取消

### 业务风险
| 风险 | 影响 | 缓解措施 |
|-----|-----|---------|
| 用户量不足，匹配慢 | 体验差 | 设置匹配等待池，定时批量匹配 |
| 圆桌质量问题 | 用户流失 | 主持人培训、问题清单优化 |
| 用户隐私泄露 | 合规风险 | 数据加密、隐私协议 |

## 9. 验收标准

### AC-001: 注册流程
**Given** 新用户打开APP
**When** 完成手机验证码验证
**Then** 
- 账号创建成功
- 进入学生证OCR页面
- 整个流程 < 60秒

### AC-002: 日历筛选
**Given** 用户已登录
**When** 设置13维度偏好
**Then**
- 日历仅显示匹配活动
- 筛选响应时间 < 500ms
- 颜色编码正确显示

### AC-003: 圆桌匹配
**Given** 用户报名圆桌
**When** 系统找到匹配度40%-70%的5人
**Then**
- 发送匹配成功通知
- 显示小组成员画像
- 显示匹配相似度

### AC-004: 圆桌讨论
**Given** 圆桌已匹配成功
**When** 会议时间到达
**Then**
- 所有成员收到开始通知
- 进入会议室
- 语音通信正常
- 问题引导正常展示

### AC-005: 认知边界图
**Given** 圆桌讨论结束
**When** 用户查看认知边界
**Then**
- 显示个人边界雷达图
- 显示小组对比
- 标注认知来源深度
