## 三、关键技术决策点（已确认）

### 决策1：引导流程顺序 ✅ 已确认

**最终方案**：简化引导流程，认知边界评估独立

```
当前实现：登录 → 产品定位 → 认知边界 → 偏好设置 → 完成
调整后：  登录 → 产品定位 → 偏好设置 → 完成

认知边界评估 → 移至"认知图谱"模块 → 用户主动点击触发
```

**理由**：认知问题清单在认知图谱中，不主动触发

### 决策2：邀请码系统实现 ✅ 已确认

**方案**：
1. 后台预设邀请码，关联群组
2. 用户注册时输入邀请码
3. 验证通过后：
   - 有关联群组 → 直接加入
   - 无关联群组 → 按偏好自动匹配

**开发方式**：并行开发（与404修复同时进行）

### 决策3：日历共享隐私 ✅ 已确认

**方案**：
- 成员接受共享 → 组长可查看"空闲/已约"状态
- 不显示具体事件内容 → 保护隐私
- 时间冲突检测 → 自动校验
