## 1. 体验北极星

> **北极星**：让每个迷茫的学生，第一次打开就能看清"原来还有这么多我不知道的选择"。

**体验原则**：
1. **零门槛入场**：学生证拍照即可，不填表、不折腾
2. **价值前置**：先给价值（日历），再要求投入（圆桌）
3. **即时反馈**：每一个操作都有肉眼可见的变化

---

## 2. 背景与动机

### 2.1 问题陈述

**目标用户**：二本三本学生，有向上发展意愿但缺乏有效信息、背景、资源。

**痛点层级**：

| 层级 | 痛点 | 表现 | 现有解决方案 |
|-----|------|------|-------------|
| 第一层 | 信息黑洞 | 不知道有什么，招聘信息满天飞但不知道哪些适合自己 | 各大招聘平台、学校就业网——信息过载，无法筛选 |
| 第二层 | 认知孤岛 | 不知道自己要什么，身边同学都迷茫，聊不出新东西 | 咨询师（贵）、学长学姐（少）、网上论坛（杂） |
| 第三层 | 路径迷雾 | 不知道该怎么走，看到岗位要求不知道该补什么 | 自己摸索、培训课程（贵且针对性差） |

### 2.2 为什么是现在

- **就业压力**：2025届毕业生预计1200万，竞争激烈
- **信息不对称**：二本三本学生缺乏有效的信息渠道和人脉
- **市场空白**：现有产品要么信息堆砌、要么只服务名校生

### 2.3 我们要解决什么

**不是**：帮学生找工作（那是招聘平台的事）

**而是**：帮学生看清自己，然后自己找到路

## 3. 目标与成功标准

### 3.1 业务目标

| 目标 | 度量方式 |
|-----|---------|
| 学生愿意来 | 注册转化率 ≥ 90% |
| 学生愿意用 | 日历访问率 ≥ 90% |
| 圆桌能开起来 | 报名率 ≥ 90%，到场率 ≥ 90% |
| 学生愿意回来 | 次日留存 ≥ 90%，7日留存 ≥ 90% |

### 3.2 止损信号

若以下任一指标连续 7 天低于 60%，触发复盘：

- 注册转化率
- 日历访问率
- 圆桌报名率
- 次日留存率

## 6. 功能需求

### 6.1 F1：注册与身份验证

#### 6.1.1 功能描述

学生通过手机号注册，上传学生证照片进行身份验证，系统OCR识别学校、专业、年级信息。

#### 6.1.2 交互流程

```
首页 → 输入手机号 → 获取验证码 → 输入验证码
→ 上传学生证照片 → OCR识别 → 确认/修改信息
→ 选择自我定位、发展方向 → 进入日历首页
```

#### 6.1.3 业务规则

| 规则 | 说明 |
|-----|------|
| 手机号唯一 | 一个手机号只能注册一个账号 |
| 学生证必传 | 用于身份验证和数据统计 |
| OCR失败处理 | 手动填写，不阻断流程 |
| 信息可修改 | 学校、专业、年级可后续修改 |

#### 6.1.4 边界条件

| 条件 | 处理方式 |
|-----|---------|
| 手机号已注册 | 提示"账号已存在，请登录" |
| 学生证模糊 | 提示重新拍照，允许跳过 |
| OCR识别错误 | 提供手动修改入口 |
| 网络超时 | 本地缓存，恢复后重传 |

#### 6.1.5 数据埋点

| 事件名 | 触发时机 | 参数 |
|-------|---------|------|
| register_start | 进入注册页 | source |
| register_code_sent | 发送验证码 | phone_masked |
| register_code_verify | 验证码验证 | success |
| register_ocr_start | 开始OCR | - |
| register_ocr_success | OCR成功 | school, major, grade |
| register_complete | 注册完成 | user_id |

### 6.2 F2：日历核心

#### 6.2.1 功能描述

按月展示招聘活动，通过颜色编码区分企业类型，支持13维度个性化筛选。

#### 6.2.2 信息结构

**日历视图**：
```
┌─────────────────────────────────────────────┐
│  ◀ 2026年3月 ▶          [筛选] [分享]      │
├─────────────────────────────────────────────┤
│  日    一    二    三    四    五    六      │
├─────────────────────────────────────────────┤
│        1     2     3     4     5     6      │
│       [2]   [5]         [3]                │
│                                              │
│   7     8     9    10    11    12    13     │
│  [1]               [4]                      │
│                                              │
│  ...                                        │
└─────────────────────────────────────────────┘

颜色图例：
🟢 互联网大厂    🟡 国企央企    🔵 外企
🟠 创业公司      🟣 事业单位    ⚪ 其他
```

**事件详情卡片**：
```
┌─────────────────────────────────────┐
│ 字节跳动 2026春招                  │
│ 🟢 互联网大厂 · 北京 · 技术岗      │
├─────────────────────────────────────┤

│ 岗位：后端开发、前端开发、测试     │
├─────────────────────────────────────┤
│ [查看详情] [加入日程] [分享]       │
└─────────────────────────────────────┘
```

#### 6.2.3 13维度筛选

| 维度 | 类型 | 说明 |
|-----|------|------|
| 地点 | 多选 | 目标工作城市 |
| 自我定位 | 单选 | 技术/产品/运营/设计/职能 |
| 发展方向 | 多选 | 管理/专家/创业/稳定 |
| 行业 | 多选 | 互联网/金融/制造/教育... |
| 平台性质 | 多选 | 大厂/外企/国企/创业公司 |
| 企业规模 | 范围 | 人数范围 |
| 企业文化 | 多选 | 扁平/层级/内卷/躺平友好 |
| 领导风格 | 多选 | 放权/指导/严格/灵活 |
| 培训项目 | 多选 | 有无、类型 |
| 加班偏好 | 范围 | 965/996接受度 |
| 假期 | 多选 | 年假、调休政策 |
| 医疗保障 | 多选 | 商业保险、体检 |
| 生育福利 | 多选 | 产假、育儿假 |

#### 6.2.4 状态机

```
┌─────────┐     点击日期      ┌─────────┐
│  月视图  │ ────────────────→ │ 日详情  │
└─────────┘                   └─────────┘
     │                             │
     │ 点击事件                    │ 返回
     ↓                             ↓
┌─────────┐                   ┌─────────┐
│ 筛选面板 │                   │  月视图  │
└─────────┘                   └─────────┘
```

#### 6.2.5 边界条件

| 条件 | 处理方式 |
|-----|---------|
| 当月无事件 | 显示"暂无符合条件的招聘活动" |
| 筛选结果为空 | 显示"放宽筛选条件试试"提示 |
| 网络错误 | 显示离线缓存数据，标记更新时间 |
| 数据加载中 | 骨架屏 + 加载动画 |

#### 6.2.6 数据埋点

| 事件名 | 触发时机 | 参数 |
|-------|---------|------|
| calendar_view | 进入日历 | month |
| calendar_filter_open | 打开筛选面板 | - |
| calendar_filter_apply | 应用筛选 | filters |
| calendar_event_click | 点击事件 | event_id, event_type |
| calendar_event_detail | 查看详情 | event_id, duration |

### 6.3 F3：圆桌讨论

#### 6.3.1 功能描述

基于用户画像匹配6人小组，进行主题讨论。完成讨论后解锁下一步，并更新认知边界图。

#### 6.3.2 交互流程

```
圆桌入口 → 查看主题 → 报名 → 等待匹配
→ 匹配成功通知 → 进入会议室 → 20个问题讨论
→ 认知图更新 → 解锁下一步
```

#### 6.3.3 会议流程

```
┌─────────────────────────────────────────────┐
│          圆桌讨论：大厂 vs 创业公司          │
├─────────────────────────────────────────────┤
│                                             │
│  [视频区域 - 6人小窗]                       │
│                                             │
│  当前问题：                                  │
│  "大厂的工作强度你能接受吗？"               │
│                                             │
│  ┌─────────────────────────────────────┐   │
│  │  匿名用户A：我听说字节很累...       │   │
│  │  匿名用户B：但是成长快啊            │   │
│  │  你：我更看重学习机会               │   │
│  └─────────────────────────────────────┘   │
│                                             │
│  [滑动条：你对这个问题了解程度？]           │
│  不了解 ◀────────────────────▶ 非常了解    │
│                                             │
│  [下一个问题]  [查看笔记]                   │
└─────────────────────────────────────────────┘
```

#### 6.3.4 匹配规则

| 规则 | 说明 |
|-----|------|
| 相似偏好 | 优先匹配筛选偏好相似的用户 |
| 时间窗口 | 报名后24小时内匹配 |
| 凑齐人数 | 满6人自动开启，不足则延期或取消 |
| 匿名机制 | 讨论过程匿名，结束后可互加好友 |

#### 6.3.5 状态机

```
          报名
            │
            ↓
┌─────────────────┐
│    等待匹配      │ ←── 24小时未凑齐 ──┐
└─────────────────┘                      │
            │                            │
            ↓ 匹配成功                   │
┌─────────────────┐                      │
│    等待开会      │ ──取消──→ 重新报名 ──┘
└─────────────────┘
            │
            ↓ 进入会议
┌─────────────────┐
│    讨论进行中    │
└─────────────────┘
            │
            ↓ 完成
┌─────────────────┐
│    已完成        │ → 解锁下一步
└─────────────────┘     更新认知图
```

#### 6.3.6 边界条件

| 条件 | 处理方式 |
|-----|---------|
| 人数不足 | 提示"等待更多伙伴加入"，24小时后取消 |
| 用户中途退出 | 标记为"中途离开"，可重新报名 |
| 网络中断 | 自动重连，显示断线提示 |
| 恶意行为 | 举报机制，人工审核 |

#### 6.3.7 数据埋点

| 事件名 | 触发时机 | 参数 |
|-------|---------|------|
| roundtable_view | 查看圆桌列表 | - |
| roundtable_signup | 报名圆桌 | topic_id |
| roundtable_match_success | 匹配成功 | group_id, topic_id |
| roundtable_join | 进入会议 | group_id |
| roundtable_question_answer | 回答问题 | question_id, score |
| roundtable_complete | 完成讨论 | group_id, duration |
| roundtable_drop | 中途退出 | group_id, reason |

### 6.4 F4：认知边界图

#### 6.4.1 功能描述

雷达图形式展示个人认知边界，颜色编码标记知识来源深度，支持时间轴查看变化轨迹。

#### 6.4.2 信息结构

```
          职业认知
             │
    ┌────────┼────────┐
    │        │        │
行业认知 ──○──────────○── 岗位认知
    │        │        │
    │        │        │
公司认知 ──○──────────○── 技能认知
             │
          发展路径

颜色图例：
🟢 来自圆桌讨论（深度）
🟡 来自日历浏览（浅层）
⚪ 系统初始（默认）
```

#### 6.4.3 知识来源标记

| 来源 | 深度 | 说明 |
|-----|------|------|
| 日历浏览 | 浅层 | 查看招聘信息获得 |
| 圆桌讨论 | 深度 | 讨论后认知变化 |
| 深度挖掘 | 更深 | 完成5W/5Whys后 |
| 实践验证 | 最深 | 完成案例实践后 |

#### 6.4.4 边界条件

| 条件 | 处理方式 |
|-----|---------|
| 新用户无数据 | 显示默认初始状态，提示"参加圆桌丰富认知" |
| 无变化 | 显示"继续探索以扩大认知边界" |
| 数据异常 | 显示最近有效数据，标记异常 |

## 8. 业务约束与风险

### 8.1 数据安全要求

| 要求 | 说明 |
|-----|------|
| 数据备份 | 每日凌晨全量备份，保留30天；每小时增量备份，保留7天 |
| 原始数据 | 第一批用户原始数据永久保留 |
| 手机号唯一 | 作为用户唯一标识 |

### 8.2 业务风险

| 风险 | 影响 | 缓解措施 |
|-----|------|---------|
| 圆桌凑不齐人 | 核心功能无法使用 | 扩大匹配时间窗口、降低匹配相似度要求 |
| 用户不回来 | 验证失败 | 通知召回、优质内容引导 |
| 数据质量差 | 筛选无效 | 多数据源、人工审核 |
