## 二、模块功能详情

### 2.1 认证模块 (Auth)

**前端路径：** `web/src/modules/auth/`
**后端路径：** `server/src/modules/auth/`

#### 功能状态

| 功能 | 前端状态 | 后端状态 | 说明 |
|------|----------|----------|------|
| 手机号验证码登录 | ✅ | ✅ | 发送验证码 + 验证码登录 |
| 密码登录 | ✅ | ✅ | 手机号 + 密码登录 |
| 密码注册 | ✅ | ✅ | 手机号 + 验证码 + 密码 + 邀请码 |
| JWT Token 管理 | ✅ | ✅ | Token 存储、刷新、过期处理 |
| 邀请码验证 | ✅ | ✅ | 注册时验证邀请码有效性 |

#### 关联模块

- **→ 邀请码模块**：注册时验证邀请码
- **→ 用户画像模块**：登录后获取/创建用户信息
- **→ 圆桌模块**：注册时根据邀请码分配群组

#### API 接口

| 方法 | 端点 | 说明 | 认证 |
|------|------|------|------|
| POST | `/api/auth/send-code` | 发送验证码 | 否 |
| POST | `/api/auth/login` | 验证码登录 | 否 |
| POST | `/api/auth/login-password` | 密码登录 | 否 |
| POST | `/api/auth/register` | 密码注册 | 否 |
| GET | `/api/auth/me` | 获取当前用户 | 是 |
| POST | `/api/auth/refresh-token` | 刷新 Token | 是 |
| POST | `/api/auth/logout` | 登出 | 是 |

## 三、完整注册流程

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                              注册流程图                                      │
└─────────────────────────────────────────────────────────────────────────────┘

用户访问注册页面
        │
        ▼
┌───────────────────┐
│  输入手机号       │
│  点击发送验证码   │
└─────────┬─────────┘
          │
          ▼
┌───────────────────┐     ┌───────────────────┐
│  POST /api/auth/  │────▶│  非生产环境返回   │
│  send-code        │     │  验证码用于测试   │
└─────────┬─────────┘     └───────────────────┘
          │
          ▼
┌───────────────────┐
│  输入验证码       │
│  输入密码         │
│  输入邀请码(可选) │
└─────────┬─────────┘
          │
          ▼
┌───────────────────────────────────────────────────────────┐
│                    POST /api/auth/register                │
│  ┌─────────────────────────────────────────────────────┐ │
│  │ 1. 验证验证码                                       │ │
│  │ 2. 检查用户是否已存在                               │ │
│  │ 3. 如果有邀请码：                                   │ │
│  │    - POST /api/invite-codes/validate (验证)         │ │
│  │    - 获取 groupId                                  │ │
│  │ 4. bcrypt 加密密码                                  │ │
│  │ 5. 创建用户                                         │ │
│  │ 6. 如果有邀请码：                                   │ │
│  │    - POST /api/invite-codes/:code/use (使用)        │ │
│  │    - 加入对应群组                                   │ │
│  │ 7. 生成 JWT Token                                   │ │
│  └─────────────────────────────────────────────────────┘ │
└─────────────────────────────┬─────────────────────────────┘
                              │
                              ▼
┌───────────────────┐
│  注册成功         │
│  返回 Token       │
│  返回用户信息     │
│  返回群组ID(如有) │
└─────────┬─────────┘
          │
          ▼
┌───────────────────┐
│  进入引导流程     │
│  → 认知边界评估   │
│  → 偏好选择       │
│  → 自动匹配圆桌   │
└───────────────────┘
```

## 六、筛选与偏好数据绑定说明

### 数据绑定关系

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                         数据绑定关系图                                       │
└─────────────────────────────────────────────────────────────────────────────┘

                    ┌─────────────────────┐
                    │   用户偏好数据       │
                    │   (数据库唯一存储)   │
                    └──────────┬──────────┘
                               │
           ┌───────────────────┼───────────────────┐
           │                   │                   │
           ▼                   ▼                   ▼
┌─────────────────┐  ┌─────────────────┐  ┌─────────────────┐
│  注册后偏好选择  │  │  个人中心偏好   │  │  左上角筛选     │
│  /preferences   │  │  /profile       │  │  /calendar      │
├─────────────────┤  ├─────────────────┤  ├─────────────────┤
│ • 必须完成      │  │ • 查看和修改    │  │ • 一次性筛选    │
│ • 保存到数据库  │  │ • 同步到数据库  │  │ • 不保存到数据库│
│ • 新用户引导    │  │ • 已登录用户    │  │ • 页面刷新重置  │
│                 │  │                 │  │ • 标注"临时筛选"│
└─────────────────┘  └─────────────────┘  └─────────────────┘
         │                   │                   │
         └───────────────────┴───────────────────┘
                       双向同步（读/写）
```

### 功能说明

1. **注册后偏好选择** `/preferences`
   - 新用户必须完成
   - 选择后保存到数据库 `user.preferences`
   - 完成后进入日历页面

2. **个人中心偏好设置** `/profile`
   - 已登录用户可查看和修改
   - 修改后同步到数据库
   - 影响推荐算法

3. **日历左上角筛选** `/calendar`
   - **一次性筛选**：页面刷新后重置
   - 默认值：读取用户偏好
   - 筛选结果：不影响用户偏好
   - UI 标注："临时筛选，不会保存"
