以下是会议记录中提取的核心功能需求，已剔除无关的讨论与废话：

### 1. 问卷与认知探索功能（摸索认知边界）

* 
**动态展示问题**：问卷中的问题必须根据用户的具体选项（如地点选“北京”或自我定位选“商人”）动态展示 。


* 
**多选题处理逻辑**：若用户进行多选（如同时选择“北京”和“上海”），每个选中的选项下方均需独立展示对应的5个特定问题 。


* 
**独立保存**：每个选项下方的问题及其回答进度，必须在后台分开独立保存 。


* 
**名称规范**：该功能模块需整合至认知图谱中，且必须严格遵循原始文档的命名，即“摸索认知边界” 。



### 2. 认知报告的更新与对比

* 
**再次编辑与更新**：用户可以多次进入报告进行查看或修改（例如修改认知了解程度的百分比） 。


* 
**自定义备注框**：修改时，每个问题下方需提供一个可选的文本输入框，供用户记录新的收获或修改原因 。


* 
**颜色区分阶段**：需使用特定的UI颜色来区分用户的修改阶段（例如：圆桌会议阶段的修改使用浅绿色框） 。


* 
**数据对比图**：增加“查看认知边界对比图”功能，用于对比用户“最初填写版本”与“最新修改版本”的数据差异，以直观体现用户的认知提升 。



### 3. 圆桌会议与群组协作流程

* 
**模块重命名**：将现有的圆桌会议相关入口重命名为“我的群组”（My Groups），作为后续所有阶段和动作的统一载体 。


* 
**团队建立与准备**：满6人后系统确认小组长，并下发系统消息提醒成员完成“摸索认知边界”的准备工作 。


* 
**日历共享与时间校验**：成员点击按钮分享日历邀请后，小组长可以随时查看所有人的日程 。


* 
**发起会议**：小组长发起会议时，填写的会议时间需自动校验其他成员的空闲时间；选填会议链接或地址 。


* 
**日程同步**：会议发起成功后，系统需将该事件自动同步并更新至所有成员的日历中 。



### 4. 主界面UI布局与个人中心

* 
**日历为主导的界面**：在主界面布局中大幅提升“日历”功能的视觉占比（约占主屏80%） 。


* 
**统一入口**：将“用户画像”、“我的群组”以及“认知边界”等功能模块作为日历界面的同级或细部入口展示，优化路径清晰度 。


* 
**学生身份认证**：“用户画像”中需增加上传照片/学生证的功能；用户只要上传学生证，系统默认认可其学生身份，无需人工额外核实 。
