# 畅选日历 - 业务概览 (biz-overview)

## 1. 一句话定位

学生职业规划领域的"张雪峰"——帮二本三本学生看清那些没人告诉他的职业真相。我们不是帮他们找工作，是帮学生看清自己，然后自己找到路。

## 2. 核心理念

### 最小完整单元 ≠ 传统MVP

- **传统MVP**：砍功能、简化、先上线再说
- **最小完整单元**：找到系统的原子，这个原子本身就是完整的、可递归生长的

**判断标准**：这个单元能否无限递归生长？如果不能，它就不是真正的最小单元。

**好的架构**：不仅要"能工作"，还要"即使失败也能获得价值"。

## 3. 目标用户

**核心人群**：二本三本学生，有向上发展意愿但缺乏有效信息、背景、资源。

**痛点层级**：

| 层级 | 痛点 | 表现 |
|-----|------|------|
| 第一层 | 信息黑洞 | 不知道有什么，招聘信息满天飞但不知道哪些适合自己 |
| 第二层 | 认知孤岛 | 不知道自己要什么，身边同学都迷茫，聊不出新东西 |
| 第三层 | 路径迷雾 | 不知道该怎么走，看到岗位要求不知道该补什么 |

## 4. 产品核心设计

### 4.1 日历核心

**功能**：按月展示招聘活动，颜色编码区分企业类型

**个性化筛选（13维度）**：
1. 地点 2. 自我定位 3. 发展方向 4. 行业 5. 平台性质
6. 企业规模 7. 企业文化 8. 领导风格 9. 培训项目
10. 加班偏好 11. 假期 12. 医疗保障 13. 生育福利

### 4.2 自我探索流程（8步解锁系统）

1. **摸索认知边界** - 20个启发式问题，滑动条评估了解程度
2. **6人圆桌会议** - 相似偏好用户组队讨论
3. **Study Buddy** - AI工具辅助探索
4. **Deep Dive** - 5W/5Whys深度挖掘
5. **Contest** - 小组竞赛
6. **Water-Testing Case Study** - 真实案例实践
7. **Focused Discussion Group** - 聚焦讨论
8. **拍卖环节** - 导师/实习机会匹配

### 4.3 认知边界对比图

- 雷达图形式展示个人认知边界
- 颜色编码标记知识来源深度
- 支持与小组成员对比
- 时间轴展示认知变化轨迹

### 4.4 社交功能

- 6人小组社群
- 日历分享（只读模式）
- 认知边界对比

## 5. MVP 1.0 范围

### 必须有的功能

| 模块 | 功能点 |
|-----|--------|
| 学生端首页 | 注册/登录、学生证拍照OCR、个人资料 |
| 日历核心 | 月份视图、事件展示、颜色编码、13维度筛选 |
| 圆桌讨论 | 匹配系统、报名入口、会议流程、实时通信、认知图更新 |
| 通知系统 | 活动提醒、圆桌通知、站内消息 |
| 认知边界图 | 个人图谱、知识来源标记、对比功能 |

### 不在MVP范围

- 雇主端
- 合作伙伴端
- 管理员端（基础后台除外）
- 完整的8步解锁（仅第1-2步）

## 7. 技术约束

### 技术路径

- **先开发Web端**，覆盖全部功能
- 后续迁移到小程序、App
- **核心风险**：数据丢失

### 数据安全要求

- 第一批用户原始数据永久保留
- 全量备份：每日凌晨，保留30天
- 增量备份：每小时，保留7天
- 手机号作为唯一标识，数据云端统一存储

### 技术选型建议

| 层级 | 推荐 |
|-----|------|
| 前端 | React |
| 后端 | Node.js |
| 数据库 | PostgreSQL |
| 实时通信 | Socket.io + WebRTC |

## 9. 商业模式

- **学生端**：免费
- **雇主端**：营销渠道¥15,000/年起，高级筛选¥120,000/年
- **合作伙伴**：AI面试官、测试等服务
